## Lessons Learned — Interactive Fact Review

**This pattern works:**
1. Export facts to JSON
2. Codex builds a web UI (Flask or http.server) that:
   - Reads the JSON, groups facts by issue type
   - Asks ONE question at a time with big mobile-friendly buttons
   - Applies answers to ALL matching facts in the database via bee CLI
   - Shows progress ("Question 3 of ~15")
3. Serve on `0.0.0.0:8765` → accessible on Tailnet at `<tailnet-ip>:8765`
4. Chris reviews fast — bulk answers, individual fact fixes, skip for edge cases

**When to use again:**
- Any time Bee facts have systematic errors (wrong names, wrong relationships, bulk misclassifications)
- Works better than CLI for Chris — visual, mobile, one-handed, no terminal
